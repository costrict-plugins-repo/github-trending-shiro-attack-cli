本版本为 **5.1.0**，核心交付 **CLI 命令行模式**，并修复多个 Gadget 兼容性问题。

**CLI 命令行模式**

- 新增 `com.summersec.attack.CLI.MainCLI` 入口，支持 6 个子命令：
  - `detect` — 探测 Shiro 框架
  - `crack` — 爆破 / 验证 Shiro Key
  - `exec` — 执行系统命令（支持自动探测 Gadget 链）
  - `memshell` — 注入内存马（哥斯拉 / 冰蝎 / 蚁剑 / NeoreGeorg / reGeorg）
  - `changekey` — 动态替换目标 Shiro Key（6 种注入变体）
  - `gui` — 启动 JavaFX 图形界面

- CLI 通过 `ConsoleTextArea` 子类化 TextArea 拦截日志输出，无需修改 `AttackService` 核心代码
- `--json` 模式下日志以结构化 JSON 输出，命令执行结果仍为原始文本
- 自动探测优先选择无 `commons-collections` 依赖的 Gadget 变体，提升首次命中率
- 详见 `skills/shiro-attack-cli/SKILL.md`

**Gadget 兼容性修复**

- 修复 `CommonsBeanutils1_183` 等 5 个 `_183` 变体中 `BeanComparator.serialVersionUID` 被错误覆写为 1.9.2 值的问题，复原为 1.8.3 的正确值 `-3490850999041592962L`。5.0.1 引入此 regression，导致针对 commons-beanutils 1.8.3 目标的所有 `_183` 链均反序列化失败

**其他**

- verifyKey 支持 AES-CBC + AES-GCM 双模式同时验证
- 修复自定义 Content-Type 时请求头出现重复 Content-Type 行
- 新增一键下载最新 Key 字典功能
- 新增 `CHANGELOG.md`、更新 README / CLAUDE.md

**升级说明**

- 推荐下载 Release 中的 **`bundle.zip`**，解压后直接运行
- CLI 启动方式：`java -cp shiro_attack-5.1.0-<jdk>.jar com.summersec.attack.CLI.MainCLI <command> [options]`
- GUI 启动方式不变：`java -jar shiro_attack-5.1.0-<jdk>.jar`
- 若已在 5.0.x 使用 GUI，可直接升级，无需额外配置
